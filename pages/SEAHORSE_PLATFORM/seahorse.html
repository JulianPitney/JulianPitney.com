Clinical Trial Data Management Platform

Project Description

This project was about building the infrastructure needed to automate and optimize a healthcare business. The company runs clinical trials for most of the major pharmaceutical companies in the world. At any one time, the company has 15-30 trials running concurrently. They've been around a while and have healthy financials, but everything was still running on paper, Excel, and a very dated SaaS system. This meant there was a lot of room for efficiency gains. They're also in the process of expanding and needed a system that could scale with them.

High-Level Goals:

  • Discover, define, and automate as many workflows as possible.
  • Digitize as much data as possible.
  • Elevate the quality of data within the organization
    • Detect and correct existing errors.
    • Guard datasets against new errors.
    • Define a data model that fits the organization's workflows and restructure existing data to fit the new data model.
    • Provide high-quality interfaces for querying datasets (Frontend GUIs for humans and APIs for software).
  • Get data and processes into the cloud securely and in compliance with PHIPA, PIPEDA, and Law 25.
  • Make frequent audits faster and easier to pass by:
    • Streamlining user management and access controls.
    • Aggregating data into fewer locations.
    • Defining digital infrastructure and processes as code.
  • Build a system that can scale to thousands of employees and hundreds of thousands of patients.
  • Build a system capable of accurate, real-time analytics.
  • Experiment with A.I tools like multi-modal LLMs, text-to-speech, speech-to-text, computer vision, etc., to try and build useful features.
  • Use automation to reduce operating cost while increasing revenue.
  • Use automation to improve patient experience and deliver a better product to customers (patients and pharmaceutical companies).

My Role:

  • Hire and lead team of 5 developers.
  • Design and implement dev + production cloud infrastructure using Terraaform, AWS and Kubernetes.
  • Build CI/CD pipelines using GitHub Actions.
  • Design ERD in DBeaver and then implement data model in Django.
  • Integrate with 3rd party clinical trials platform (CRIO) using their API.
  • Explore and experiment with various A.I tools (OpenAI API, langchain, AWS A.I functions, various other tools)
  • Propose, negotiate and oversee cybersecurity and regulatory audit by 3rd party company (PacketLabs in Toronto).
  • Earn buy-in across the organization by delivering results and advocating for our work using effective communication.
  • Motivate the team and get them the resources they need.
